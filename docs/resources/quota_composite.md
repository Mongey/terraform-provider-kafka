# kafka_quota_composite (Resource)

Manages Kafka quota paths that require two entity components — any `/config/users/<x>/clients/<y>` path. For single-entity paths, use [`kafka_quota`](quota.md) instead.

The primary use case is IAM-authenticated clients (e.g. AWS MSK IAM auth), where the Kafka `user` principal is an unpredictable session ID. `/config/users/<default>/clients/<client-id>` (Kafka quota precedence #4) sidesteps this: `user` matches any IAM session and `client-id` targets a stable, app-controlled value.

## Example Usage

```terraform
# Cluster-wide default for all users (precedence #6)
resource "kafka_quota" "default_all_users" {
  entity_type = "user"
  config = {
    "producer_byte_rate" = "1048576"
    "consumer_byte_rate" = "2097152"
  }
}

# Higher limit for a specific IAM service client (precedence #4)
resource "kafka_quota_composite" "iam_service" {
  entity {
    type = "user"
    # no name = entity-default (matches any user, including IAM sessions)
  }
  entity {
    type = "client-id"
    name = "my-iam-service"
  }
  config = {
    "producer_byte_rate" = "4194304"
    "consumer_byte_rate" = "8388608"
  }
}
```
